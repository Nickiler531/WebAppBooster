What is WebAppBooster?

WebAppBooster is a free Android app you can download from Google's Play Store. It gives web applications you visit access to your smartphone's sensors and locally stored data. The benefit is that WebAppBooster enriches a web application by enabling functionality that ordinarily only a native app can do.

How does is work?

WebAppBooster runs a service in the background of your smartphone. When you surf to a web page with your browser, the web application can contact this background service to request access to your devices capabilities.

Is it safe?

Yes. You will have to grant permissions to a web application before it can access any sensors or local data on your device. This is identical to when you install an app from the Google Play Store: before Android will allow installation of an app, you first have to grant it the requested permissions. WebAppBooster will do the same for web apps.

Where can I get it?

WebAppBooster is an Open Source application. You can download it from the Google Play Store for your Android device. Once WebAppBooster is installed, you can try out a little demo.

Developer

New plugins have to be registered in res/xml/plugins.xml. A plugin has a name that corresponds with the name of the Java class implementing the plugin. A plugin responds to one or more actions. Before a web app can access the plugin, the user has to first grant the appropriate permission. The following excerpt of plugin.xml shows how to add a plugin called NewPlugin that responds to actions FOO_COMMAND_1 and FOO_COMMAND_2 and requires the user to grant a permission called FOO_PERMISSION:

<plugins>
    <plugin class="AuthenticationPlugin"
            actions="REQUEST_AUTHENTICATION|AUTHENTICATE"/>
    <plugin class="PermissionsPlugin"
            actions="REQUEST_PERMISSIONS"/>
    ...
    <plugin class="FooPlugin"
            actions="FOO_COMMAND_1|FOO_COMMAND_2" permission="FOO_PERMISSION"/>
</plugins>

The following code excerpt shows the implementation of the plugin NewPlugin. Plugins have their own lifecycle. Each web app is serviced by its own instance of a plugin. When the connection is opened, onCreate() is invoked. When the web app terminates the connection, onDestroy() is invoked. Whenever the web app calls an action, the execute() method of the appropriate plugin is invoked. Method execute() runs in the context of the WebAppBooster background service. If the plugin needs to render a user interface, it first needs to call method runInContextOfProxyActivity(). This will in turn invoke the method callbackFromProxy(): 

public class FooPlugin extends Plugin {

    @Override
    public void onCreate(String origin) {
        // ...
    }

    @Override
    public void onDestroy() {
        // ...
    }

    @Override
    public void execute(int requestId, String action, JSONObject request) {
        // ...
        // runInContextOfProxyActivity();
    }

    @Override
    public void callbackFromProxy() {
        // ...
    }
}

Once the new plugin is implemented and registered, it can be accessed from JavaScript by using the helper function WebAppBooster._sendRequest(). This function takes three arguments: (1) a JSON object that must have an action attribute whose value matches the declaration in plugins.xml. (2) a callback function that is invoked when the web app receives a response. (3) a boolean flag that indicates if the action will send one (0) or multiple replies (1). The latter is used for actions that trigger a continuous stream of sensor data (e.g., gyroscope): 

var WebAppBooster = {
    // ...
    foo1Command: function(cb) {
        var req = {action: "FOO_COMMAND_1"};
        this._sendRequest(req, cb, 0);
    },
    foo2Command: function(cb) {
        var req = {action: "FOO_COMMAND_2"};
        this._sendRequest(req, cb, 0);
    }
};

Frequently Asked Questions

  1. Is WebAppBooster free?
    Yes. WebAppBooster is released under the BSD Open Source license.
  2. Is WebAppBooster also available for iOS?
    No. Apple restricts the kind of services that can run in the background. WebAppBooster would not comply to Apple's license agreement.
  3. What kind of Web API is supported?
    Currently WebAppBooster only supports access to the users contact list and the gyroscope. Additional plugins can easily be implemented.
  4. What permissions does WebAppBooster require?
    WebAppBooster acts as a gateway to the phones data and sensors. As such, it requires Android permissions to access to access data such as contacts and sensors such as GPS. However, a web app will only be given access to those sensitive resources if the user explicitly granted the appropriate permissions.
